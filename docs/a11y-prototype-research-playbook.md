# A11y Prototype Research Playbook

프로토타입 단계에서 접근성 리서치를 **가볍게 시작**하기 위한 실행용 플레이북입니다.

## 1) 언제 이 플레이북을 쓰나

- Figma만으로는 실제 보조기기/키보드 사용성을 확인하기 어려울 때
- 출시 직전이 아니라, 요구정의/프로토타입 단계에서 위험을 먼저 줄이고 싶을 때
- 팀이 작아서 복잡한 리서치 프로세스 대신 **작고 반복 가능한 루틴**이 필요할 때

---

## 2) 30~60분 실행 루틴

### Step A. Survey (10~15분)

목표: 이해도/혼란 지점을 빠르게 수집

- [ ] 핵심 과업 1개를 짧게 설명했다
- [ ] 상태 전달(색만 vs 텍스트+아이콘) 이해 여부를 물었다
- [ ] 확대(200%)/다크모드에서 읽기 어려운 지점을 물었다

### Step B. Co-design (15~20분)

목표: 사용자와 함께 흐름/문구를 정리

- [ ] 핵심 플로우 1개(예: 검색→상세→제출)를 선정했다
- [ ] 버튼/입력 라벨 문구를 함께 정리했다
- [ ] 탭 순서(첫 포커스, 모달 닫힘 후 복귀)를 합의했다

### Step C. Rapid HTML (15~25분)

목표: 브라우저에서 키보드/포커스 동작을 현실적으로 검증

- [ ] 테스트 페이지에서 Tab/Shift+Tab 이동을 확인했다
- [ ] `:focus-visible` 가시성을 확인했다
- [ ] `label`/`aria-label` 접근성 이름을 확인했다
- [ ] 200% 확대 시 잘림/겹침이 없는지 확인했다

권장 파일: `docs/a11y-rapid-harness.html`

## 4) 운영 팁 (가볍게 유지)

- 한 번에 모든 화면을 하지 말고, **핵심 플로우 1개**만 선택
- 회차마다 이슈를 1~2개만 확실히 줄이는 것을 목표로 설정
- 결과는 PR/WORKLOG에 남겨 다음 회차에서 이어서 개선
